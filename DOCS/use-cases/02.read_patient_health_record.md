### 2. Read Patient Health Record

**Actor:** Healthcare Provider, Patient (with proper authorization)

**Description:** Retrieve and view patient health record information.

**Preconditions:**

- User is authenticated and authorized to access the record
- Patient record exists in the system

**Main Flow:**

1. User searches for patient by ID, name, or other identifier
2. System retrieves record from primary database or read replica (load balancing)
3. System checks user's authorization level
4. System returns patient record with appropriate data visibility
5. User views patient information, medical history, medications, etc.
6. System logs the access for audit purposes

**Postconditions:**

- Patient record is displayed to authorized user
- Access is logged in audit trail

**Exception Flow:**

- If patient not found, system displays "Patient not found" message
- If user lacks authorization, system denies access and logs attempt
- If primary database unavailable, system reads from replica
- If all databases unavailable, system displays error and suggests retry
